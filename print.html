Usage

NoticeMe can be used with event or exports, both are available

TriggerEvent("noticeme:Notify", "Hello, World!")
-- or
exports.noticeme:Notify("Hello, World")

Available functions

NameDescriptionParameters
NotifyDefaultmessage/options: If the argument is of type string then it will be the message printed. Alternatively a table can be used to specify options
InfoAlias for Notify with set optionsmessage: The message to print
WarningAlias for Notify with set optionsmessage: The message to print
ErrorAlias for Notify with set optionsmessage: The message to print

Options

NameTypeDescription
messagestringThe message to print
timeoutintegerHow long(ms) the notification should be visible for
typestringThe type of the notification
audiobooleanWhether to play audio or not
imagestringA base64 encoded image to display instead of icons

Default options

NameDefault
message"Null Content"
timeout2000 for Info, 2500 for Warning, 3000 for Error
type"info" for Info, "warn" for Warning, "error" for Error
audiofalse for Notify, true for the Aliases
imagenull

Examples

Notify

TriggerEvent("noticeme:Notify", {
    message = "Hello, World!",
    timeout = 2000, -- 2 seconds
    audio = true, -- use notification sound
    type = "info", -- use the info colors and icons
})

Customization

Currently there aren't any variables to quickly configure the resource but it's planned. Here you can find some tips on how to manually customize the resource.

Offset from the top

To change the offset from the top of the screen you need to modify line 17 of ui.js

More notification types

To add your own notification types you need to:

  • Create css class and modify any css attributes you want:
.notification.--myType {
    border-left: 5px solid blue;
}
  • Add an icon for this class in the icons dictionary in ui.js
  • Add a title for this class in the titles dictionary in ui.js
  • Add a sound file in the sounds directory (or copy-paste the existing sound and rename it to match your class name)
